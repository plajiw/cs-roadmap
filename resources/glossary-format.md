# Guia de Formato - Glossary.md

## Objetivo
Manter um padrão consistente para registrar termos e conceitos fundamentais de forma clara, objetiva e didática.

## Estrutura Padrão para Cada Termo

```markdown
## [Termo em Português]

**Definição**: [Uma linha, concisa e direta. O essencial do conceito]

**Origem**: [Etimologia, contexto histórico ou razão do nome, se relevante]

**Explicação Técnica**: [Detalhamento mais aprofundado, contexto de aplicação, detalhes práticos]

**Relacionado**: [Outros termos vinculados, separados por vírgula]

---
```

## Diretrizes

- **Definição**: Máximo 1-2 linhas. Responda: "O que é?"
- **Origem**: Opcional, mas recomendado. Ajuda a fixar o conceito.
- **Explicação Técnica**: Expanda com contexto, aplicações e detalhes. Máximo 3-4 linhas.
- **Relacionado**: Liste apenas termos que já existem no glossário ou que serão adicionados em breve.
- **Linguagem**: Português do Brasil, técnico, objetivo. Evitar exemplos lúdicos.
- **Separadores**: Use `---` entre termos para clareza visual.

## Exemplo Completo

```markdown
## Cache

**Definição**: Camada de armazenamento rápido que mantém cópias de dados frequentemente acessados.

**Origem**: Do francês "caisse" (caixa), referindo-se a um lugar seguro para guardar algo.

**Explicação Técnica**: Mecanismo de otimização que reduz latência armazenando dados em mídia mais rápida (geralmente memória RAM). Usado em múltiplas camadas: processadores (L1/L2/L3), sistemas operacionais, aplicações web e bancos de dados. Requer estratégia de invalidação para manter consistência.

**Relacionado**: Bootstrap, Kernel, Memória RAM

---
```

## Ordenação

Organize os termos alfabeticamente para facilitar buscas.
